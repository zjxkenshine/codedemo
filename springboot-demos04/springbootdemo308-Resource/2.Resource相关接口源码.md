# 1.Resource接口：
```java
public interface Resource extends InputStreamSource {
    boolean exists();   // 确定此资源是否实际以物理形式存在。
    default boolean isReadable(){}  // 
    default boolean isOpen() {}  // 如果为true，则不能多次读取InputStream，必须将其读取并关闭以避免资源泄漏
    default boolean isFile() {}  // 获取该资源的File文件
    URL getURL() throws IOException;  // 返回该资源的URL句柄
    URI getURI() throws IOException;  // 获取uri
    File getFile() throws IOException;  // 获取该资源的File文件
    default ReadableByteChannel readableChannel() throws IOException {} //返回一个ReadableByteChannel 。预计每次调用都会创建一个新通道                                                        
    long contentLength() throws IOException; //确定此资源的内容长度
    long lastModified() throws IOException; //确定此资源的最后修改时间戳
    //根据相对路径创建资源
    Resource createRelative(String relativePath) throws IOException; 
    String getFilename();   //文件名
    String getDescription(); // 获取资源的描述
}
```

# 2.InputStreamSource接口：
```java
public interface InputStreamSource {
    //为底层资源的内容返回一个InputStream 
    //每次调用都返回一个新流
    InputStream getInputStream() throws IOException;
}
```

# 3.ContextResource接口：
```java
public interface ContextResource extends Resource {
    //返回封闭的“上下文”中的路径
	String getPathWithinContext();
}
```

# 4.WritableResource接口:
```java
public interface WritableResource extends Resource {
    //是否可写
    default boolean isWritable() { }
    //为底层资源返回一个OutputStream ，允许（覆盖）写入其内容
    OutputStream getOutputStream() throws IOException;
    //返回一个可写通道
    default WritableByteChannel writableChannel() throws IOException { }
}
```

## 5.HttpResource接口：
```java
public interface HttpResource extends Resource {
    //提供一个Http头用于当前资源返回
	HttpHeaders getResponseHeaders();
}
```
